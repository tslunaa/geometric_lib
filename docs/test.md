# *Лабораторная работа №4 — Unit-тестирование геометрической библиотеки*

## Отчёт + Полный комплект тестов

---

# **1. Общая информация**

Данная лабораторная работа посвящена написанию **unit-тестов** для библиотеки геометрических функций, созданной в ходе лабораторной работы №1.
Тестируются функции вычисления **площади и периметра**:

* круга (circle.py)
* прямоугольника (rectangle.py)
* квадрата (square.py)
* треугольника (triangle.py)

Все тесты пишутся с использованием стандартного Python-модуля **unittest**.

---

# **2. План тестирования (по требованиям)**

## **2.1 Цели и задачи тестирования**

**Цель:** проверить корректность вычислений площадей и периметров всех геометрических фигур библиотеки geometric_lib.

**Основные задачи:**

* удостовериться, что функции возвращают корректные математические результаты;
* проверить работу функций на валидных данных;
* убедиться, что функции корректно обрабатывают крайние условия (ноль, большие числа);
* обеспечить автоматическое тестирование через unittest.

---

## **2.2 Описание тестируемого продукта**

Тестируемая библиотека включает 4 модуля:

| Модуль       | Функции                     | Назначение                        |
| ------------ | --------------------------- | --------------------------------- |
| circle.py    | area(r), perimeter(r)       | Площадь и длина окружности        |
| rectangle.py | area(a,b), perimeter(a,b)   | Площадь и периметр прямоугольника |
| square.py    | area(a), perimeter(a)       | Площадь и периметр квадрата       |
| triangle.py  | area(a,h), perimeter(a,b,c) | Площадь и периметр треугольника   |

Содержимое модулей взято из загруженных файлов.

---

## **2.3 Область тестирования**

Будут протестированы:

* функции `area()` и `perimeter()` всех фигур;
* расчёт корректных значений на валидных данных;
* нулевые и большие значения;
* симметричные/равные стороны;
* классические тесты (3/4/5 треугольник, квадрат 10×10 и т. п.).

---

## **2.4 Стратегия тестирования**

Методы тестирования:

* **Unit-тесты** — основной метод;
* **Функциональное тестирование** — проверка математической корректности;
* **Позитивные тесты** — корректные входные данные;
* **Крайние случаи** — нулевые длины, большие числа.

Типы тестов:

* **Тестирование функций** — каждая функция будет проверяться изолированно;
* **Модульное тестирование** — проверяются все функции каждого файла;
* **Автоматизация** — запуск через `python -m unittest`.

---

## **2.5 Критерии приёмки**

Тест считается успешным, если:

* 100% тестов прошли успешно;
* все функции возвращают корректные значения формул;
* тестирование запускается командой:

```
python -m unittest discover
```

или

```
python -m unittest tests/test_figures.py
```

---

## **2.6 Ожидаемые результаты**

По итогам тестирования должны быть получены:

* отчёт о прохождении всех тестов;
* статистика (количество тестов, время выполнения);
* протокол с ошибками (если обнаружены);
* наличие тестов в истории коммитов в репозитории.

# 4. Как запускать тесты

```

```

